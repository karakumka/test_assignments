# Тестовое задание "Редактирование изображений по промпту"

## 1. Загрузка данных

Для тестового задания взят [датасет](https://huggingface.co/datasets/NarchAI1992/Roof_house) с картинками домов с сайта HuggingFace. Его размер удовлетворяет требованию "30-50 изображений". Для ускорения обработки файлов был проведен ресайз картинок: максимальная высота/ширина равняется 256, размер второго измерения высчитывается с сохранением пропорций.

## 2. Подготовка промптов

Для решения задачи была собрана корзина промптов, к каждому из которых добавлен промпт по удалению водяных знаков и контактной информации.

## 3. Выбор модели и запуск инференса

Изначально тестировалась модель `Qwen/Qwen-Image-Edit`, однако локально не удалось добиться стабильного инференса из-за высоких требований к ресурсам. Поэтому для получения конечного результата была использована более легкая модель `black-forest-labs/FLUX.2-klein-4B`. Выбор обусловлен типом решаемой задачи "Image-to-Image"; среди доступных вариантов данной категории модели были отсортированы [по количеству загрузок](https://huggingface.co/models?pipeline_tag=image-to-image&sort=downloads).

Промпт, применяемый к изображению, выбирался случайным образом через генерацию целого числа и использованию его в качестве индекса в списке промптов.

Количество шагов генерации задано 20, так как обработка каждого из файлов занимала значительное количество времени и ресурсов. При 20 шагах среднее время обработки файла - 2 минуты 30 секунд (32ГБ RAM, 8ГБ VRAM).

##  4. Сохранение результата

Исходный HF-датасет был преобразован в файлы с расширением `.png` и сохранен на диске. Результаты генерации также сохранены на диске в формате `.png`. 

По окончании генерации сформирован JSON-файл с метаданными. Для каждого изображения сохранялись:

- `input_id` - имя исходного файла;
- `prompt` - промпт, использованный для редактирования;
- `output_id` - имя сгенерированного файла.

```json
[
    {
        "input_id": "image_11.png",
        "prompt": "Add a broken glass effect to the picture. Remove the watermarks from the picture and any contact information.",
        "output_id": "image_11_upd.png"
    }
]
```

## 5. Техническая валидация

На этапе валидации JSON-метаданных проверялись следующие условия:

- Каждая запись содержит обязательные поля `input_id`, `prompt` и `output_id`, а значения полей не являются пустыми;
- В файле JSON отсутствуют дубликаты `input_id` и `output_id`;
- Исходный и сгенерированный файл действительно существуют на диске и доступны для открытия;
- Высота/ширина изображений не равны нулю.

Если хотя бы одно из условий нарушается, скрипт валидации выводит сообщение "Validation failed" и список найденных ошибок. Если ошибок не найдено, выводится сообщение "Validation passed".
